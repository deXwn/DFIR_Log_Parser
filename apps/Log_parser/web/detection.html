Workflow

Suggested operator path

  • Point the runner at the same root path used during search.
  • Start with a preset, then narrow it into a custom rule if needed.
  • Persist hits, review false positives, and export only validated findings.
Signal Quality

Reduce noisy hits

  • Use one sharply defined rule before combining several broad detections in the same run.
  • Keep false-positive review close to the first run so rule tuning stays grounded in evidence.
  • Persist only the runs that produce investigation-grade signal; discard exploratory noise.
Rule Design

Keep rules focused

  • Use thresholds and time windows for burst-style activity.
  • Prefer strong field conditions over broad regex-only matches.
  • Save environment-specific rules once noise has been reduced.
